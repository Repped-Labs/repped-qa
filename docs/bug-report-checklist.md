# Bug Report Checklist

Run this before you submit every bug report. If any item is unchecked, fix it first.

- [ ] Title is specific and does not start with "It" or "The"
- [ ] Location is filled in — the specific page, section, or feature where the bug occurs
- [ ] Staging URL is included — the exact Circle page where you found it
- [ ] Steps to Reproduce are numbered — someone who has never used this widget could follow them and reach the same bug
- [ ] Expected Result is written — what should have happened
- [ ] Actual Result is written — what actually happened
- [ ] Severity is chosen — Critical / High / Medium / Low — with a one-sentence reason
- [ ] Workaround is filled in — describe it if one exists, or write "None"
- [ ] Environment is filled in — browser and operating system
- [ ] Screenshot is attached — showing the moment the bug occurs

> A bug report that cannot be reproduced is not a bug report. It is a note to yourself.
