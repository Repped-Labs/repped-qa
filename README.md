# Repped Labs — QA Board

This is the public QA repo for Repped Labs. If you are a QA apprentice, this is where you work.

---

## What this is

Repped Labs builds interactive widgets embedded inside Circle communities. You are blackbox testing those widgets in a live staging environment — as a real logged-in Circle member, with no access to the source code.

Your job: find bugs, file clear reports, and verify fixes. Real work. Real product. Real experience.

---

## Getting started

1. **Read the workflow doc first** → [docs/qa-apprentice-workflow.md](docs/qa-apprentice-workflow.md)
2. **Go to the board** → [QA Board](https://github.com/orgs/Repped-Labs/projects/7)
3. **Pick up a task** from the Ready for QA column
4. **Get your staging access** from Trinity — you need a Circle community invite

That's it. The board tells you what to test. The workflow doc tells you how.

---

## How to file a bug

Click **New Issue** → select **Bug Report**. Fill in every field. Link it back to the QA task that surfaced it.

---

## Labels

| Label | Meaning |
|---|---|
| `status: ready for qa` | Unclaimed — pick it up |
| `status: in qa` | You're on it |
| `status: qa passed` | Signed off |
| `status: qa failed` | Bugs found, waiting on fix |
| `type: bug` | Bug report |
| `type: qa task` | Something to test |
| `priority: high` | Test this first |
| `priority: medium` | After high priority |
| `priority: low` | When capacity allows |
| `stale` | No activity for 3+ days — needs attention |
